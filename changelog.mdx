<Update label="May 29, 2026" description="Weekly release">

## New features

- **Auto-completing DCA schedules.** When registering a DCA schedule, you can now set a `totalOrders` value that defines how many executions should run before the schedule finishes on its own. No more manual cleanup of long-running schedules. See [Wealth Engine — DCA](/features/wealth-engine#dollar-cost-averaging-dca).

## Updates

- **DCA lifecycle visibility.** DCA schedules now expose explicit lifecycle statuses (`ACTIVE`, `COMPLETED`, `CANCELLED`) so your agent can report progress like "12 of 30 DCA orders completed." Progress fields `ordersExecuted` and `totalOrders` are returned by [`GET /v1/actions/wealth/intents`](/api-reference/actions/wealth-intents).
- **Clearer DCA parameter reference.** The [Register DCA Schedule](/api-reference/actions/wealth-dca) endpoint documentation now reflects all current parameters and their behavior end-to-end.

</Update>
